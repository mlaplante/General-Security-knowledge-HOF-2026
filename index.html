Phishing Attacks

The Most Common Cyber Threat

What is Phishing?

Phishing is a social engineering attack where attackers impersonate trusted entities to steal sensitive information.

Goal: Trick you into revealing passwords, credit card numbers, or installing malware

Types of Phishing

  • Email Phishing: Mass emails pretending to be from legitimate companies
  • Spear Phishing: Targeted attacks on specific individuals
  • Whaling: Attacks targeting executives and high-profile targets
  • Smishing: Phishing via SMS/text messages
  • Vishing: Voice phishing via phone calls

Red Flags to Watch For

  • Urgent or threatening language
  • Poor grammar and spelling
  • Generic greetings ("Dear Customer")
  • Suspicious sender addresses
  • Unexpected attachments
  • Requests for sensitive info
  • Mismatched URLs
  • Too good to be true offers

Real Example

Subject: URGENT: Your account will be closed!

Dear Valued Customer,

We detected suspicious activity on your account. Click here immediately to verify your identity or your account will be permanently suspended within 24 hours.

üö© Red Flags: Urgency, generic greeting, suspicious link

How to Protect Yourself

  • Hover over links before clicking
  • Verify sender email addresses carefully
  • Never provide credentials via email
  • Use email filtering and anti-phishing tools
  • When in doubt, contact the company directly
  • Report suspicious emails to IT/Security

Social Engineering

Hacking the Human Element

What is Social Engineering?

Social engineering is the art of manipulating people into divulging confidential information or performing actions that compromise security.

"It's easier to hack a person than to hack a system."

Common Tactics

Pretexting
Creating false scenarios

Baiting
Offering something enticing

Urgency
Creating time pressure

Authority
Impersonating leaders

Tailgating
Physical access tricks

Dumpster Diving
Searching discarded info

Real-World Scenario

The CEO Email Scam

An employee receives an urgent email appearing to be from the CEO:
"I'm in a meeting with investors and need you to wire $50,000 to this account immediately. Don't discuss this with anyone - it's confidential. Time sensitive!"

This uses authority + urgency + secrecy to bypass normal procedures.

Defense Strategies

  • Verify identities through multiple channels
  • Follow established security procedures
  • Be skeptical of unusual requests
  • Never let urgency override security
  • Educate yourself on manipulation tactics
  • Trust your instincts - if it feels wrong, verify

How Malware Spreads

  • Email Attachments: Infected files disguised as documents
  • Malicious Links: URLs leading to infected downloads
  • Software Vulnerabilities: Exploiting unpatched systems
  • USB Drives: Physical media carrying infections
  • Compromised Websites: Drive-by downloads

Protection Measures

  • Enable automatic updates
  • Regular backups (3-2-1 rule)
  • Avoid downloading from untrusted sources
  • Scan all external drives before use
  • Use application whitelisting

Two-Factor Authentication

Adding an Extra Layer of Security

What is 2FA/MFA?

Authentication using two or more factors to verify identity:

Something you know - Password or PIN
Something you have - Phone, token, smart card
Something you are - Fingerprint, face, voice

Why 2FA Matters

99.9%
Reduction in account compromise with MFA enabled

Even if your password is compromised, attackers still can't access your account!

Types of 2FA

SMS Codes

Text message verification

Authenticator Apps

Google/Microsoft Authenticator

Hardware Tokens

YubiKey, physical devices

Biometrics

Face ID, Touch ID

Best Practices

  • Enable 2FA on all critical accounts
  • Prefer authenticator apps over SMS when possible
  • Keep backup codes in a secure location
  • Use hardware tokens for highest security
  • Never share 2FA codes with anyone
  • Review and update authentication methods regularly

Data Privacy

Protecting Your Digital Footprint

What is Data Privacy?

Data privacy is about controlling who has access to your personal information and how it's used.

Personal data includes: names, addresses, emails, phone numbers, IP addresses, location data, browsing history, biometric data, and more

Privacy Risks

  • Data Breaches: Unauthorized access to databases
  • Identity Theft: Stolen personal info used fraudulently
  • Tracking: Companies monitoring your online behavior
  • Data Mining: Aggregating data to profile you
  • Social Media Exposure: Oversharing personal details
  • Third-Party Sharing: Your data sold to others

Privacy Protection Steps

  • Review and adjust privacy settings on all platforms
  • Limit information shared on social media
  • Use VPN on public Wi-Fi networks
  • Clear cookies and browsing data regularly
  • Use privacy-focused browsers (Brave, Firefox)
  • Read privacy policies before agreeing
  • Opt out of data sharing when possible

Professional Data Privacy

Work Data Requires Extra Care
  • Never share proprietary company information
  • Protect customer/client data as if it were your own
  • Follow company data handling policies
  • Encrypt sensitive data in transit and at rest
  • Use approved tools and platforms only
  • Report data incidents immediately

Incident Response

What to Do When Compromised

If You Click a Phishing Link

  1. Don't Panic - Act quickly but calmly
  2. Disconnect - Turn off Wi-Fi or unplug ethernet
  3. Don't Enter Info - Close the page immediately
  4. Scan System - Run antivirus/anti-malware
  5. Change Passwords - Start with email, then others
  6. Report It - Notify IT/Security team

If You're Hacked

Signs of Compromise: Unexpected password resets, unfamiliar activity, locked accounts, strange emails sent from your account
  • Change all passwords immediately
  • Enable 2FA if not already active
  • Review recent account activity
  • Check for unauthorized access/devices
  • Notify contacts if account was compromised
  • Monitor credit reports and financial accounts

Ransomware Attack Response

NEVER PAY THE RANSOM - No guarantee of file recovery
  1. Isolate infected systems immediately
  2. Report to IT/Security and law enforcement
  3. Do not delete or modify anything
  4. Restore from clean backups
  5. Use decryption tools if available
  6. Document everything for insurance/legal

Building an Incident Response Plan

1. Preparation - Have tools and contacts ready
2. Detection - Identify and confirm the incident
3. Containment - Limit the damage
4. Eradication - Remove the threat
5. Recovery - Restore normal operations
6. Lessons Learned - Improve for next time

Network Security

Protecting Your Connections

Understanding Network Threats

Networks are prime targets for attackers seeking to intercept data, inject malware, or gain unauthorized access.

Man-in-the-Middle
Intercepting communications

Evil Twin
Fake Wi-Fi hotspots

Packet Sniffing
Capturing network traffic

DNS Spoofing

Public Wi-Fi Dangers

43%
of cyberattacks target public Wi-Fi users
  • Unencrypted connections expose your data
  • Attackers can see your browsing activity
  • Credentials can be stolen without your knowledge
  • Malware can be injected into downloads

Securing Your Network

  • Always use VPN on public networks
  • Verify network names before connecting
  • Use HTTPS for all sensitive transactions
  • Disable automatic Wi-Fi connection
  • Turn off file sharing on public networks
  • Keep firewall enabled at all times
  • Use strong WPA3 encryption at home

Home Network Best Practices

Router Security

  • Change default admin password
  • Update firmware regularly
  • Disable WPS
  • Hide SSID (optional)

Network Segmentation

  • Separate guest network
  • IoT devices on isolated network
  • Work devices on separate VLAN
  • Monitor connected devices

Zero Trust Architecture

Never Trust, Always Verify

What is Zero Trust?

Zero Trust is a security framework that assumes no user or system should be trusted by default, even inside the network perimeter.

"Never trust, always verify"
Core Principle: Verify every user, device, and transaction - every single time

Zero Trust Principles

Verify Explicitly
Authenticate everything

Least Privilege

Assume Breach

Implementing Zero Trust

  • Identity verification at every access point
  • Multi-factor authentication everywhere
  • Micro-segmentation of networks
  • Continuous monitoring and analytics
  • Context-aware access policies

  • Data encryption and protection

Secure Software Development

Building Security In, Not Bolting It On

Why Secure Development Matters

70%
of security vulnerabilities are in application code

Security must be integrated throughout the software development lifecycle, not added as an afterthought.

Common Vulnerabilities (OWASP Top 10)

  • Injection: SQL, command, LDAP injection attacks
  • Broken Authentication: Weak session management
  • Sensitive Data Exposure: Unencrypted data
  • XML External Entities (XXE): XML processor attacks
  • Broken Access Control: Unauthorized access
  • Security Misconfiguration: Default settings
  • Cross-Site Scripting (XSS): Injected scripts
  • Insecure Deserialization: Object injection

Secure Coding Practices

  • Input validation and sanitization
  • Output encoding
  • Parameterized queries (prevent SQL injection)
  • Proper authentication and authorization
  • Secure session management
  • Encryption of sensitive data
  • Error handling without information leakage
  • Regular security testing and code reviews
